常用的CSS hack技巧

之前公司做的项目需要运行在客户的ie8环境下，开发环境在谷歌
所以经常就是这边开发，测试的时候要开ie8来看是否一致，交互时候完成业务
有时候真的遇到一些让人摸不着头脑的兼容问题，这类问题一般是CSS在ie8下属性不支持
也有属性支持，但是表现不一致。还有一些对JS内置对象属性的支持不一致了。
这里主要把一些常用的hack技巧总结总结，避免以后自己忘记了又得百度找半天

hack技巧的分类

  • CSS属性前缀法
  • 选择器前缀法
  • IE条件注释法（即HTML头部引用if IE）Hack

CSS属性前缀法

属性前缀法是在CSS样式属性名前加上一些只有特定浏览器才能识别的hack前缀，以达到预期的页面展现效果 在标准模式中
“-”减号和下划线“_”是IE6专有的hack
“*, +”IE6， IE7生效
“\9” IE6/IE7/IE8/IE9/IE10都生效
“\0” IE8/IE9/IE10都生效，是IE8/9/10的hack
“\9\0” 只对IE9/IE10生效，是IE9/10的hack

选择器前缀法

*html *前缀只对IE6生效
*+html *+前缀只对IE7生效
@media screen\9{...}只对IE6/7生效
@media \0screen {body { background: red; }}只对IE8有效
@media \0screen\,screen\9{body { background: blue; }}只对IE6/7/8有效
@media screen\0 {body { background: green; }} 只对IE8/9/10有效
@media screen and (min-width:0\0) {body { background: gray; }} 只对IE9/10有效
@media screen and (-ms-high-contrast: active), (-ms-high-contrast: none) {body { background: orange; }} 只对IE10有效

条件注释法

只在IE下生效
只在IE6下生效
只在IE6以上版本生效
只在IE8上不生效
非IE浏览器生效
相应的方法可右键浏览器查看源代码中这部分的注释

个人总结

CSS属性前缀法的hack对于书写顺序很重要，排在最前面的要适用性广，识别能力强
这点参考JS的类型检测的鸭式辩形

选择器前缀法是针对一些页面表现不一致或者需要特殊对待的浏览器

条件注释法主要的用途在于文档中根据不同的浏览器来决定是否引入相应的JS文件

Css hack能不用就不用，不仅CSS文件丑陋，对于后期开发者的维护成本也高
对于日益接近统一的浏览器，我想往后的兼容问题将会越来越少，所以hack也随之更加少用
当然前提是项目针对的客户群体不同，如果客户电脑是个老古董，停留在XP，用着老式IE8 IE7 IE6
那还是很头痛的
